## Release v1.1.9

**Release Date:** April 27, 2026

Патч-релиз относительно **v1.1.8**: корректный заголовок **Routing** для Happ (deeplink), единый источник **VLESS flow** из настроек инбаунда, доработки Hysteria в панели, дашборд и мониторинг CPU.

### Подписка и client routing

- **HTTP `Routing`:** значение теперь в формате **`{prefix}{base64}`** (по умолчанию `happ://routing/add/…`), с пресетами **incy** / **sharx** / **custom** на профиль — в духе конструктора в панели.
- **Нормализация:** сырые Base64, устаревший `happ://add/…` с JSON-пейлоадом и ручные заголовки приводятся к ожидаемому виду; **`RoutingPayloadBase64ForSubscription`** оставлен для сценариев, где нужен только сегмент Base64.

### VLESS

- **Flow** задаётся только в **настройках инбаунда** (`settings.clients[].flow`); при сохранении клиента и в Xray/подписке используется это значение, а не поле клиента из формы.

### Панель (UI)

- **Дашборд:** клик по **RAM** открывает историю памяти; **swap** и **диск** больше не открывают график CPU.
- **Инбаунды (Hysteria / Hysteria2):** расширенные поля транспорта/TLS (в т.ч. uTLS fingerprint), пресет потока при смене протокола.

### Прочее

- **Telegram CPU alerts:** выборка загрузки за **5 с** вместо блокирующей минуты; **cooldown 5 мин** между повторными уведомлениями при превышении порога.
- **inboundDefaults:** правки дефолтов/пресетов для Hysteria-семейства.

### 📦 Upgrade from v1.1.8

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.1.9`** / **`latest`** и пересоздайте контейнер панели.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```

Полный список изменений: `git log v1.1.8..v1.1.9` / `git diff v1.1.8 v1.1.9`.
