## Release v1.3.6

**Release Date:** May 12, 2026

Релиз относительно **v1.3.5**: исправления для **Telemt (MTProto)** в режиме **multi-node** — когда клиенты ходят через прокси на **worker-узле**, панель теперь корректно получает **счётчики трафика и онлайн**, не отбрасывает опрос узла при остановленном Xray, и показывает **активные сессии (IP)** даже при ошибках Xray API. В **standalone** эти данные уже подмешивались локально (`MergeLocalTelemtTrafficIntoXrayStats`, `CollectLocalTelemtOnlineSessions`); на воркере учёт Telemt шёл через `MergeTelemtIntoNodeStats` / `CollectOnlineSessionsForUser`, но несколько ранних выходов из HTTP-обработчиков и панельного `CollectNodeStats` обнуляли или блокировали этот путь.

Миграций БД **нет**.

### Bug Fixes

- **Worker `GET /api/v1/stats`** (`node/api/server.go`)
  - Раньше при **`ErrXrayNotReady`** (ядро Xray не запущено или API ещё не готово) обработчик отвечал **503** до вызова **`MergeTelemtIntoNodeStats`**, из-за чего панель не видела MTProto-трафик и онлайн, хотя Telegram через Telemt работал.
  - Теперь в этом случае возвращается **200** с пустой статистикой Xray и тем же слиянием **Telemt** (локальный control API sidecar’ов на `127.0.0.1`), как после успешного опроса Xray.

- **Панель `CollectNodeStats`** (`web/service/node.go`)
  - Раньше при **`GET /health`** с `xrayRunning: false` опрос **`GET …/api/v1/stats`** для узла **полностью пропускался**, даже если **Telemt** на узле был запущен (`telemtRunning` / `telemtCount` из health).
  - Теперь пропуск только если **и Xray выключен, и Telemt неактивен**; если Telemt активен — запрос статистики выполняется (в паре с правкой worker `/stats` выше).

- **Worker `GET /api/v1/user-online-sessions`** (`node/api/server.go`)
  - Раньше любая ошибка Xray **кроме** `ErrXrayNotReady` (например отсутствие **`statsUserOnline`** в политике, сбой gRPC `GetUserOnlineIPList`) приводила к **500** **до** вызова **`CollectOnlineSessionsForUser`**, поэтому в multi-node **не отображались активные сессии** по Telemt, тогда как для других протоколов IP часто шли из Xray.
  - Теперь такая ошибка логируется как **warning**, список сессий Xray обнуляется, и **Telemt** по-прежнему дополняется из control API (`/v1/users/…`).

### Improvements

- Логи: при частичном отказе Xray для user-online-sessions сообщение уровня **Warning** с префиксом `user-online-sessions: xray query failed (continuing with telemt): …` для диагностики без обрыва ответа клиенту панели.

### Operational Notes

- Обновите **и панель, и worker-образы** до **1.3.6**, чтобы заработали все три правки (панель — `CollectNodeStats`; worker — `/stats` и `/user-online-sessions`).
- Если после обновления счётчики или сессии по Telemt пустые, проверьте на узле: sidecar Telemt запущен, в сгенерированном TOML включён **`[server.api]`** (по умолчанию localhost), **email клиента** в панели совпадает с пользователем в Telemt для API.
