## Release v1.4.2

**Release Date:** May 28, 2026

Релиз **v1.4.2** относительно **v1.4.1**: патч-релиз с двумя исправлениями для **Telemt MTProto** в multi-node Prometheus-merge: (1) онлайн-статус и «**Last connection node**» теперь корректно отражают активные telemt-сессии; (2) счётчик «**Total Received**» (Down) на странице **Inbounds** перестал залипать на `0 B`.

Миграций БД **нет**.

---

### Bug Fixes

- **Telemt MTProto: пользователи не попадают в `OnlineClients` при Prometheus-merge** (`node/telemt/stats_merge.go`)
  - **Причина:** в v1.3.9 добавлена ветка `mergeTelemtPrometheusCounters`, которая поллит `/metrics` сидекара и сливает per-user counters в `ClientTraffic`. Эта ветка возвращает `true` (короткое замыкание — legacy JSON-путь `/v1/users` пропускается), но **не вызывает `addOnline(user)`** — функция даже не получала этот замыкатель. В legacy-пути `addOnline` вызывается при `CurrentConnections > 0`.
  - **Следствие:** воркер в ответе `GET /api/v1/stats` отдавал корректный `ClientTraffic` (траффик и `lastOnline` обновлялись), но **пустой `OnlineClients`** для telemt-клиентов. Панель `CollectNodeStats` собирала `onlineClientsMap` только из этого списка → `setPanelOnlineClients` не содержал telemt → в `ClientCardView.IsOnline = false`, счётчик «Online users» = 0, колонка «Last connection node» = «—».
  - **Фикс:** проброс `addOnline` в `mergeTelemtPrometheusCounters` и вызов `addOnline(user)` при положительной дельте (`dFrom > 0 || dTo > 0`). Логика зеркалит `onlineFromTraffic` в `node/xray/manager.go::GetStats` для Xray-юзеров.

- **Telemt MTProto: `Total Received` инбаунда всегда 0 при Prometheus-merge** (`node/telemt/stats_merge.go`)
  - **Причина:** Prometheus-ветка эмитила **два** `xray.Traffic` entry на тик: один с реальным тегом `{Tag: tag, Up: tagUp, Down: 0}` и второй — с **поддельным** тегом `{Tag: tag + "_up", Up: 0, Down: tagDown}`. Панельный `CollectNodeStats` маппит теги в `inboundId` через `tagToInboundId` — поддельный тег `*_up` не существует, поэтому Down молча падал в ветку `logger.Debugf("Unknown tag in traffic: ...")` и не учитывался в `inboundTrafficMap`.
  - **Следствие:** на странице **Inbounds** для telemt-инбаундов `Total Sent/Received` всегда показывало `X GB / 0 B`. В БД (`inbounds.down`, `inbounds.all_time`) Down-направление не аккумулировалось.
  - **Фикс:** один entry с реальным тегом инбаунда и обоими направлениями — `{Tag: tag, Up: tagUp, Down: tagDown}`. Это симметрично тому, как Xray /stats возвращает inbound traffic (один тег, два поля).

### Improvements

- В отсутствие реальной активности (delta == 0) пользователь не помечается онлайн — поведение симметрично Xray-ветке и legacy JSON-ветке telemt.

### Operational Notes

- Обновление затрагивает только **ноду** (`sharxnode`) — панель (`sharx`) обновления не требует, но рекомендуется обновлять обе до одинаковой версии.
- После раскатки воркера в multi-node:
  - Счётчик «Online users» в `Clients` начинает корректно отражать telemt-сессии.
  - В строке клиента появляется «Last connection node» с именем узла, через который сейчас идёт трафик.
- Активные IP-сессии в модалке (`/panel/client/sessions/{id}`) для telemt дополнительно требуют включённого `[server.api]` в telemt TOML на узле (это уже было документировано в `release-notes/v1.3.6.md`). Текущий фикс затрагивает только агрегированный online/last-node, не IP-список.

### 📦 Upgrade from v1.4.1

1. Бэкап `.env` (миграции БД не требуются).
2. Обновите образ **`sharxnode`** до **`1.4.2`** / **`latest`** на всех узлах. Образ **`sharx`** обновления не требует, но рекомендуется привести к той же версии.
3. Проверьте:
   - На странице `Clients`: активный telemt-клиент через 5–30 секунд после трафика показывается «Online», в колонке «Last connection node» появляется имя ноды.
   - На странице `Inbounds`: для telemt-инбаунда колонка `Total Sent/Received` показывает оба направления (Received перестаёт быть `0 B`).

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
