## Release v1.2.4

**Release Date:** April 28, 2026

Патч-релиз относительно **v1.2.3**: добавлена статистика User-Agent по HWID (по клиенту и на дашборде), реализован раздел управления geofile-ассетами, расширено логирование и потоковая доставка логов, обновлены переводы и UI.

### Key Features

- **HWID User-Agent analytics:** распределение User-Agent по HWID для каждого клиента и сводная статистика на общем дашборде.
- **Geofile assets management:** отдельный раздел для работы с `geoip.dat` / `geosite.dat` (загрузка, скачивание по URL, применение, откат, хранение версий).
- **Logging and streams:** добавлены доработки логирования и websocket-стрим логов для панели/нод.
- **Localization updates:** расширены переводы и текстовые ресурсы в panel/web локалях.

### What's New

- **Backend / DB:** новая миграция `0035_geofile_assets`, сервис `geofile_asset`, API-эндпоинты и оркестрация применения geofile на локальной панели и нодах.
- **Panel UI:** обновлены `ClientsPage`, `DashboardPage`, `XrayPage`, `SettingsPage`, добавлен маршрут `panel/xray/geo`.
- **Node / Core integration:** доработаны node API и менеджер Xray для применения geofile и сопровождения логов.
- **Docs / API:** актуализирована документация API под новые сценарии.

### 📦 Upgrade from v1.2.3

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.2.4`** / **`latest`** и пересоздайте контейнеры панели/ноды.
3. Убедитесь, что миграции применились (включая таблицу geofile assets), после старта проверьте новый раздел geofile и HWID User-Agent статистику.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
