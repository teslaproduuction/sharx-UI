## Release v1.1.8

**Release Date:** April 27, 2026

Патч-релиз относительно **v1.1.7**: маршрутизация клиента в ответе подписки, улучшения конструктора routing в настройках страницы подписки, правки UI обновлений.

### Функции

- **Подписка (HTTP-заголовки):** для первого **inline**-профиля из `routing.profiles` в конфиге страницы подписки в ответ подписки автоматически добавляются **`Routing`** (Base64 от компактного JSON) и **`Routing-Enable: 1`**. Профили только с URL не подставляются (без сетевых запросов на каждый `GET`). Заголовки из **`responseRules.extraHeaders`** по-прежнему могут переопределить эти значения.
- **Конструктор client routing (панель):** убрана внешняя ссылка на сторонний док; добавлено read-only поле **сгенерированной deep link** и кнопка копирования рядом с ним; уточнён текст про формат `…/routing/add/{Base64}`.

### Улучшения UI

- **Окно «Об обновлении»:** на узких экранах заметки релиза не дублируют боковую колонку; на `lg+` сохраняется разметка с боковой панелью. Для `Modal` добавлены опции **`closeOnEscape`**, **`lockBodyScroll`**, **`portalClassName`** (в т.ч. для корректного поведения при стэке диалогов).

### 📦 Upgrade from v1.1.7

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.1.8`** / **`latest`** и пересоздайте контейнер панели.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
