## Release v1.3.7

**Release Date:** May 12, 2026

Релиз относительно **v1.3.6**: исправление регресса по **Telemt (MTProto)** — **`telemt_secret`** в таблице **`client_inbound_mappings`** больше **не пересоздаётся** при каждом сохранении клиента, если в запросе меняется только набор инбаундов (например добавлен **Hysteria2**, а MTProto-инбаунд уже был). Раньше `UpdateClient` при непустом **`InboundIds`** удалял **все** маппинги клиента и вызывал **`AssignClientToInbounds`**, из‑за чего для Telemt каждый раз генерировался новый 32-символьный hex-секрет и ломались уже выданные пользователю **`tg://proxy`** ссылки. Поведение приведено к той же идее, что и для **WireGuard**: стабильные секреты/ключи после первого назначения.

Миграций БД **нет**.

### Bug Fixes

- **`UpdateClient` + назначение инбаундов** (`web/service/client.go`)
  - **Было:** при `client.InboundIds != nil` выполнялось `DELETE` всех строк `client_inbound_mappings` для клиента, затем **`AssignClientToInbounds`** создавал записи заново → для протокола **`telemt`** снова вызывался **`GenerateTelemtSecretHex()`** для каждого Telemt-инбаунда в списке.
  - **Стало:** добавлена **`SyncClientInboundAssignments`**: удаляются только маппинги, которых нет в новом списке; для существующих пар (клиент, инбаунд) строка **сохраняется** (включая **`telemt_secret`** и `id`), при необходимости обновляется только **`sort_order`** под порядок в массиве; новые инбаунды добавляются одной вставкой с новым секретом **только** для вновь подключённого Telemt-инбаунда.
  - **`AssignClientToInbounds`** по-прежнему используется при **создании** клиента (`CreateClient`), где полная вставка маппингов корректна.

### Operational Notes

- Обновление только **панели** до **1.3.7** достаточно (worker не менялся в этом релизе).
- У пользователей, которые уже успели сохранить клиента и получить **новый** секрет до патча, в подписке/ссылках останется последний выданный секрет; после обновления панели дальнейшие правки списка инбаундов секрет **не будет** меняться сам по себе.

1. Бэкап БД и `.env`.
2. Образ панели **`1.3.7`** / **`latest`** (миграции не требуются).
3. Проверить: клиент с Telemt + добавление другого инбаунда и сохранение — **`telemt_secret`** в БД для существующего маппинга **не меняется** (можно сравнить до/после одним `UPDATE` клиента в UI).
