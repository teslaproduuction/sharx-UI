## Release v1.0.1

**Release Date:** March 4, 2026

### 🐛 Bug Fixes

#### Client Management
- **Fixed issue #30**: Fixed reset flow parameter - now allows clearing the `flow` field when editing a client
- **Fixed client modal**: Fixed issue where removing all inbound assignments from a client didn't save properly to the database
- **Improved inbound search**: Added smart search functionality in client modal for assigned inbounds - now searchable by name (remark), port, ID, and protocol (fixes issue #26)

#### Nodes Management
- **Fixed issue #28**: Added sorting functionality to nodes table - default sort by ID (ascending), sortable by ID, Name, and Address
- **Added Xray version column**: Added "Xray Version" column to nodes table displaying the running Xray core version on each node

#### UI Improvements
- **Fixed duplicate icon**: Removed duplicate Xray configuration icon on overview page
- **Fixed ID column**: Prevented "ID" column header text from wrapping to second line in nodes table

### ✨ New Features

#### Happ Encryption
- **Local Happ encryption**: Implemented local Happ encryption using RSA-4096 with PKCS1 v1.5 padding
  - Supports `happ://crypt4/` format
  - Uses public key from `@kastov/cryptohapp` library for compatibility
  - No longer requires external API calls for encryption

### 🔄 Changes

- Improved error handling and logging for client inbound assignments
- Enhanced WebSocket updates to include Xray version information for nodes
- Improved JSON request handling for empty arrays in client updates

### 📋 Technical Details

- **Encryption**: Changed from OAEP to PKCS1 v1.5 padding for Happ encryption compatibility
- **Database**: Improved handling of empty arrays in client inbound assignments
- **Frontend**: Enhanced search functionality in client modal with filter options
- **Backend**: Added Xray version retrieval from nodes via `/api/v1/status` endpoint

### 🐳 Docker Images

Images are available in Harbor:

- **sharx**: `registry.konstpic.ru/sharx/sharx:1.0.1` / `registry.konstpic.ru/sharx/sharx:latest`
- **sharxnode**: `registry.konstpic.ru/sharx/sharxnode:1.0.1` / `registry.konstpic.ru/sharx/sharxnode:latest`

### 📦 Upgrade Instructions

Update your `docker-compose.yml`:

```yaml
services:
  sharx:
    image: registry.konstpic.ru/sharx/sharx:1.0.1
  sharxnode:
    image: registry.konstpic.ru/sharx/sharxnode:1.0.1
```

Or use `latest` tag for automatic updates:

```yaml
services:
  sharx:
    image: registry.konstpic.ru/sharx/sharx:latest
  sharxnode:
    image: registry.konstpic.ru/sharx/sharxnode:latest
```
