## Release v1.3.4

**Release Date:** May 8, 2026

Релиз относительно **v1.3.3**: разделён жизненный цикл **Xray и Telemt** на worker-узлах (отдельные stop/restart, `stop-xray` больше не гасит sidecar’ы), панель хранит и показывает **состояние Telemt по узлам**, расширены **статус панели** и **node API** (счётчики Telemt, health), добавлена **трассировка хешей** при выдаче/применении worker-конфига (**SHA-256** готового JSON и **хеш core profile**), улучшена синхронизация **Xray core config profiles** с worker’ами (в т.ч. узлы только с mapping профиля, смена профиля у узла без «конфликтной» ошибки), ускорено применение изменений для **Telemt-инбаундов**, для **VLESS/Trojan** добавлено редактирование **`settings.fallbacks`** в форме инбаунда, обновлены **переводы** и мелкие UI-правки. Пример **`node/docker-compose.yml`** очищен от захардкоженных секретов (пустые `PANEL_URL` / `SECRET_KEY` — подставить локально).

Миграции БД: **`0040_node_telemt_state.sql`** — колонка **`telemt_state`** в **`nodes`** (`running` | `stopped` | `unknown`).

### New Features

- **Telemt vs Xray на worker (node agent)**
  - **`POST /stop-xray`** останавливает только Xray; Telemt **не** останавливается автоматически (`node/api/server.go`).
  - Новые endpoints: **`POST /stop-telemt`**, **`POST /restart-telemt`** (replay последнего успешного Telemt snapshot после `Apply`) (`node/api/server.go`).
  - В **`GET /health`** и **`GET /api/v1/status`** добавлены **`telemtRunning`**, **`telemtCount`** (`node/api/server.go`, `node/telemt/manager.go` — `RunningCount`, snapshot для restart).

- **Управление сервисами с панели (узлы и standalone)**
  - API узлов: **`POST /panel/api/node/stopTelemt/:id`**, **`stopXray/:id`**, **`restartXray/:id`**, **`restartTelemt/:id`** (`web/controller/node.go`, `web/service/node.go`).
  - При **выключении узла** в панели явно вызывается остановка Telemt на worker, затем Xray (`web/controller/node.go`).
  - Локальный режим (не multi-node): **`POST /panel/api/server/stopTelemtService`**, **`restartTelemtService`** (`web/controller/server.go`, `web/service/server.go`, `web/service/telemt_panel.go`).

- **Состояние Telemt в БД и статусе панели**
  - Модель **`Node.TelemtState`** и константы **`NodeTelemt*`** (`database/model/model.go`).
  - **`RefreshNodeTelemtStateFromWorker`** обновляет поле из статуса worker (`web/service/node.go`).
  - В **`GetStatus`**: блок **`telemt`** для локальных sidecar’ов и агрегат **`nodesTelemt`** (сколько узлов в каком `telemt_state`, только `enable=true`) (`web/service/server.go`).

- **Хеши конфигурации worker ↔ панель**
  - Сборка: **`BuildWorkerXrayConfigForNodeWithMeta`** возвращает JSON и **SHA-256 hex** от `ConfigJson` выбранного **core config profile** (если профиль задействован) (`web/service/xray.go`).
  - В теле **`apply-config`** на node (wrapped JSON): опциональные **`coreProfileHash`**, **`expectedConfigSha256`**; в ответе — эхо хеша профиля и **`configSha256Match`** при переданном expected (`node/api/server.go`, `web/service/node.go` — `ApplyWorkerConfigMeta`).
  - **Pull worker config** (`pull-xray-config`): в JSON-ответе **`coreProfileHash`**, **`configSha256`** (`web/controller/api.go`).

- **Xray core config profiles**
  - В выдаче профилей в API добавлено вычисляемое поле **`configHash`** (от содержимого `ConfigJson`) (`web/service/xray_core_config_profile.go`, `database/model/model.go`).
  - **Назначение узлов профилю**: если узел был на другом профиле, старая привязка снимается, узел переносится на выбранный профиль **без ошибки «конфликт»**; после операции — точечный push конфига на затронутые узлы (`web/service/xray_core_config_profile.go`, `web/controller/xray_core_config_profile.go`, `web/docs/API.md`).
  - После обновления / сброса профиля / смены default — **push** worker-конфига на релевантные узлы (`web/controller/xray_core_config_profile.go`).
  - Выбор профиля при сборке worker JSON: приоритет **явных `profile_node_mappings`**, затем инбаунды / default (`web/service/xray.go`).
  - Логика **кого пушить** в multi-node расширена: узлы с инбаундами **и/или** привязкой профиля (`multiWorkerNodeIDsNeedingPush`, `MergeUniquePositiveInts`, `ApplyWorkerConfigToNodeIDs`, и т.д. в `web/service/xray.go`).

- **Инбаунды Telemt: быстрее применяются изменения**
  - При мутациях инбаунда/клиента протокола **Telemt** планируется **`RestartXrayAsync(false)`** вместо отложенного `SetToNeedRestart` (`web/controller/inbound.go`).

- **VLESS / Trojan: `fallbacks` в UI**
  - Типы строк, парсинг и сборка **`settings.fallbacks`** в настройках инбаунда (`panel/lib/inboundDefaults.ts`, изменения в **`InboundsPage`** и связанных компонентах).

### Improvements

- **Страница узлов (`NodesPage`)**: отображение и действия вокруг Telemt/Xray согласованы с новым API (см. diff `panel/components/NodesPage.tsx`).
- **Дашборд**: доработки в связке с расширенным статусом/метриками (`panel/components/DashboardPage.tsx`).
- **UI kit**: правки **`checkbox-field`**, **`select-native`**, экспорт компонентов (`panel/components/ui/*`).
- **Редакторы subscription page**: мелкие правки (`AddToAppEditor`, `InstallationGuideEditor`, `InstallationGuideBlock`).
- **Прочее:** правки **`OutboundsBuilder`**, **`TableViewer`**, **`ClientsPage`**, **`XrayCoreConfigProfilesPage`**; косметика **`third_party/telemt-sharx/prebuilt/*/.gitkeep`**.

- **Локализация:** обновлены `panel/public/locales/*.json` и `web/translation/*.toml` (в т.ч. заметное обновление `translate.ru_RU.toml`).

### Operational / Breaking Notes

- После обновления панели примените миграцию **`0040`** — иначе поле **`telemt_state`** отсутствует.
- Worker-ноды желательно обновить вместе с панелью, чтобы работали **`/stop-telemt`**, **`/restart-telemt`** и расширенный status/health.
- Интеграции, парсящие ответ **pull-xray-config**, могут использовать новые поля **`coreProfileHash`** и **`configSha256`**.
- Пример **`node/docker-compose.yml`**: задайте **`PANEL_URL`** и **`SECRET_KEY`** (или pairing-секрет) в окружении; в репозитории оставлены пустые значения как шаблон.
