## Release v1.2.5

**Release Date:** April 29, 2026

Патч-релиз относительно **v1.2.4**: удален legacy web UI ассет-пайплайн (Vue/Ant static bundle), добавлены настройки отображения IPv6 в дашборде, улучшены UX и управление сборкой remark-модели, а также обновлен flow sidecar-updater.

### Key Features

- **Legacy web cleanup:** из репозитория удалены устаревшие `web/assets` и legacy HTML-шаблоны после миграции на panel-приложение.
- **IPv6 dashboard toggle:** добавлена серверная и UI-настройка `enableIPv6` для включения/отключения определения и отображения публичного IPv6.
- **Remark model UX update:** переработан конструктор remark-модели (drag-and-drop порядка полей, быстрые кнопки добавления, более понятные подсказки).
- **Updater feedback flow:** улучшен UX запуска sidecar-обновления с отдельным модальным состоянием и прогресс-индикатором до авто-reload.

### What's New

- **Backend / Settings:** расширены `AllSetting` и `SettingService` новым флагом `enableIPv6`, обновлена логика `ServerService.GetStatus()` для условного получения IPv6.
- **Panel UI:** обновлены `SettingsPage`, `DashboardPage`, `PanelHeaderAppMeta`, добавлен виджет `user_agent` в layout с миграцией сохраненных раскладок.
- **Web cleanup:** удалены legacy JS/CSS/vendor-ассеты и старые HTML-компоненты, оставшиеся от предыдущего web-слоя.

### 📦 Upgrade from v1.2.4

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.2.5`** / **`latest`** и пересоздайте контейнеры панели/ноды.
3. После старта проверьте, что:
   - в настройках доступен переключатель IPv6 для дашборда;
   - блок IPv6 на дашборде корректно скрывается/показывается;
   - новый release-notes отображается в UI апдейтера.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
