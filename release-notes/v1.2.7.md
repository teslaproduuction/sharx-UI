## Release v1.2.7

**Release Date:** April 30, 2026

Патч-релиз относительно **v1.2.6**: исправлена работа протокола `mixed` в multi-node режиме — аккаунты всех назначенных клиентов теперь корректно отражаются в Xray конфиге, переработана форма редактирования `mixed` инбаундов (убраны лишние поля и шаги).

### Bug Fixes

- **[#59](https://github.com/konstpic/SharX/issues/59) Mixed protocol: все назначенные клиенты теперь в `accounts`**
  - В multi-node режиме при назначении второго (и последующих) клиентов на `mixed` инбаунд их учётные данные теперь корректно попадают в `accounts` Xray конфига на всех нодах.
  - Исправлена логика `onlySettingsChanged` в `UpdateInbound`: ранее сравнение происходило с уже перезаписанным значением, из-за чего fast API update (`update-inbound`) не срабатывал и изменения не синхронизировались.
  - Исправлена функция `mergeFirstClientIntoSettings` на фронтенде: ранее при любом редактировании `mixed` инбаунда `accounts` перезаписывался одним аккаунтом из формы, затирая остальных.

### Improvements

- **Mixed инбаунд: убрана форма аутентификации**
  - Шаг "Authentication" и поля `Account user` / `Password` скрыты для протокола `mixed` — аккаунты управляются исключительно через назначение клиентов.
  - При создании нового `mixed` инбаунда `settings` инициализируется как `{"auth":"noauth","udp":true}` — без захардкоженного аккаунта.
  - При редактировании `mixed` инбаунда `settings` теперь передаётся из БД без изменений (со всеми назначенными аккаунтами).

- **Mixed инбаунд: упрощена форма транспорта**
  - `mixed` выделен в отдельный transport mode — только TCP, без WebSocket (SOCKS5/HTTP клиенты не поддерживают WS транспорт).
  - Убраны кнопки WebSocket и селект Network из шага "Transport & security" для `mixed`.
  - При загрузке существующего `mixed` инбаунда network принудительно нормализуется в `tcp`.

### 📦 Upgrade from v1.2.6

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.2.7`** / **`latest`** и пересоздайте контейнеры панели/ноды.
3. После старта проверьте:
   - что при назначении нескольких клиентов на `mixed` инбаунд все их учётные данные отображаются в Xray config preview;
   - что подключение каждого клиента работает со своими уникальными кредами через SOCKS5 и HTTP.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
