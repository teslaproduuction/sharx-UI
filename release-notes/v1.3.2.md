## Release v1.3.2

**Release Date:** May 6, 2026

Релиз относительно **v1.3.1**: добавлена интеграция **Telemt (MTProto sidecar)** в панель и node-агент, расширены API и UI для MTProto/секретов, добавлена синхронизация онлайн-сессий и мердж трафика Telemt в статистику панели, обновлены Docker/CI сценарии релиза, в репозиторий добавлен upstream-код `third_party/telemt-sharx`.  
Миграции БД: **`0039_client_inbound_telemt_secret.sql`** (новое поле `telemt_secret` в `client_inbound_mappings`).

### New Features

- **Telemt sidecar integration (panel + node)**
  - Новый менеджер Telemt на стороне node: запуск/остановка, применение payload-конфигов, сбор online sessions и объединение статистики (`node/telemt/*`).
  - На стороне панели добавлен standalone-контур для локального применения Telemt при single-node режиме (`web/service/telemt_panel.go`, `web/service/telemt_config.go`).
  - Добавлена синхронизация Telemt-сессий между узлами и сервисные точки входа для обновления состояния (`web/service/telemt_session_sync.go`, `sub/telemt_tg.go`).

- **MTProto per-client secret**
  - Новая миграция **`0039`** добавляет поле `telemt_secret` на уровне связи клиент↔inbound.
  - Обновлены модели/сервисы/API для чтения и сохранения секрета, включая обработку на backend и node (`database/model/model.go`, `web/service/inbound.go`, `node/api/server.go`).

- **UI updates for MTProto/Telemt**
  - Расширен `InboundsPage`: поля и логика для Telemt/MTProto, редактирование и валидация секретов, обновления поведения форм.
  - Внесены изменения в `ClientsPage`, `HostsPage` и `panel/lib/inboundDefaults.ts` для поддержки новых настроек.

- **Bundled Telemt upstream**
  - Добавлен каталог `third_party/telemt-sharx` с исходниками, prebuilt-бинарями и документацией для интеграции Telemt в поставку.

### Improvements

- **Статистика и онлайн-сессии**
  - Дельты Telemt мержатся в Xray-образную статистику, чтобы трафик и сессии отражались консистентно в панели.
  - Добавлена сборка online IP sessions для MTProto через Telemt Control API.

- **Node config/runtime**
  - Обновлены `node/main.go`, `node/configpull/pull.go`, `xray/api.go`, `web/service/xray.go` для корректного жизненного цикла Telemt вместе с Xray-контурами.

- **Docker and release pipeline**
  - Обновлены `Dockerfile`, `node/Dockerfile`, `DockerInit.sh`, `node/docker-compose.yml`.
  - Расширен workflow `harbor-release` и добавлен скрипт `scripts/build-telemt-sharx.sh` для сборки/публикации Telemt-компонентов.

### Documentation

- Обновлены разделы API и контроллеры, связанные с Telemt/MTProto (`web/docs/API.md`, `web/controller/api.go`, `web/controller/inbound.go`).

### Breaking / Operational Notes

- В релиз включён очень крупный third-party пакет `third_party/telemt-sharx` (существенный рост размера репозитория и образов).
- Для корректной работы новых полей и API необходимо применить миграцию **`0039`**.

### 📦 Upgrade from v1.3.1

1. Сделайте **бэкап** базы данных и `.env`.
2. Поднимите панель и node на версии **`1.3.2`** (образы **`1.3.2`** / **`latest`**).
3. Убедитесь, что применена миграция **`0039_client_inbound_telemt_secret.sql`**.
4. После старта проверьте:
   - редактирование MTProto inbound’ов и сохранение `telemt_secret`;
   - запуск Telemt sidecar и видимость online sessions;
   - корректность суммарной статистики трафика в панели.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
