## Release v1.2.6

**Release Date:** April 29, 2026

Патч-релиз относительно **v1.2.5**: переработан публичный subscription page (упрощен visual flow и расширено управление действиями), добавлены DB-backed UI preferences (тема/язык/layout), улучшена multi-node синхронизация inbound и сбор статистики, обновлены Telegram-уведомления, а также удален неиспользуемый cache-слой.

### Key Features

- **Public subscription UX refresh:** блок `links-list` переведен в режим "header actions only", улучшены светлые пресеты/контрасты, унифицированы кнопки Copy/QR и fallback-брендинг.
- **Persistent UI preferences in backend:** `panelTheme`, `panelLang`, `dashboardWidgets`, `hideSecAlert`, `clientsTablePrefs` теперь хранятся в settings DB и доступны через API.
- **Multi-node traffic/accounting fixes:** доработан сбор node stats с fallback-поведением для временно рассинхронизированных inbound mappings, добавлен update inbound-трафика и `last connected node` для online-клиентов.
- **Hot-sync inbound assignment:** перераспределение inbound по нодам теперь сразу синхронизируется через node API (add/remove без полного рестарта), с fallback на full apply-config.
- **Telegram bot notifications upgrade:** добавлена поддержка animated/custom emoji, расширена локализация сообщений и введены отдельные уведомления о derived-состоянии клиента (ACTIVE/DISABLE/LIMITED/EXPIRED).
- **Subscription API enhancements:** в публичной выдаче добавлены encrypted deeplink-поля для `happ://` и `v2raytun://`, а `appMeta` обогащается server-side UI pref значениями.

### What's New

- **Subscription page / Panel UI:**
  - удален визуальный рендер `links-list` карточек, но сохранены его конфиги для header actions;
  - поведение Copy/QR в header теперь управляется блоком и корректно отключается при его отсутствии;
  - расширены light-mode color presets (`default`, `midnight`, `ember`, `boreal` и др.) с более читаемыми поверхностями и акцентами;
  - унифицированы стили action-кнопок/иконок для шапки и QR-modal.
- **Settings and UI preferences:**
  - добавлены API-роуты `POST /panel/api/inbounds/ui/get` и `POST /panel/api/inbounds/ui/set`;
  - введен allowlist для ключей UI preferences на сервисном уровне;
  - `SettingsPage` переключена на чтение/запись темы через backend UI prefs;
  - поле языка Telegram-бота заменено на нормализованный dropdown с полным набором поддерживаемых локалей;
  - локализованы LDAP labels (`Bind DN`, `Base DN`) через i18n ключи.
- **Node and XRAY operations:**
  - в node manager добавлено детальное structured-логирование `ApplyConfig` (payload size, inbound delta, skip/restart decision);
  - улучшен `CollectNodeStats`: учет inbound traffic delta, обновление inbound counters, soft-fallback при отсутствии inbound mappings;
  - в panel runtime записывается `lastConnectedNode` по online-клиентам;
  - при выключении multi-node или отсутствии нод очищаются runtime-карты connected-node состояния.
- **Inbound assignment sync:**
  - `AssignInboundToNodes` теперь фиксирует old/new mappings и выполняет hot-sync изменений;
  - для новых назначений используется `UpdateInboundOnNode`, для снятых - `RemoveInboundFromNode`;
  - при stopped XRAY на worker добавлен fallback на `BuildWorkerXrayConfigForNode + ApplyConfigToNode`.
- **Backend cleanup and consistency:**
  - удален устаревший пакет `web/cache` и middleware-кэш;
  - из `SettingService` убрана cache-зависимость для single/all/default settings чтений;
  - упрощены пути чтения settings с явным DB source of truth.
- **Public API and app metadata:**
  - `publicAppMeta` теперь учитывает сохраненные `panelLang`/`panelTheme` (если доступны);
  - публичная подписка возвращает encrypted deeplink URLs для совместимых клиентов.
- **Traffic and client services:**
  - расширены обновления клиентского трафика и связанной статистики в multi-node сценариях;
  - улучшена консистентность runtime online-клиентов/нод при частичных сбоях node poller.
- **Telegram bot and localization:**
  - динамическая загрузка набора animated emoji (`NEWSEMOJI`) + fallback custom IDs;
  - унифицирован рендер emoji через `<tg-emoji>` для сообщений в HTML parse mode;
  - переписаны inbound/group notifications на i18n-ключи;
  - добавлены новые translation entries в `web/translation/*` и sync-проверка через `web/i18n_toml_test.go`.

### 📦 Upgrade from v1.2.5

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.2.6`** / **`latest`** и пересоздайте контейнеры панели/ноды.
3. После старта проверьте:
   - что тема/язык панели и layout-виджеты сохраняются между сессиями;
   - что публичная subscription page корректно отдает header Copy/QR согласно конфигу;
   - что в multi-node режиме обновляются inbound traffic counters и online/last-node данные;
   - что Telegram-бот отправляет обновленные сообщения с корректной локализацией.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
