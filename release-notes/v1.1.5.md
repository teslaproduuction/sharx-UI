## Release v1.1.5

**Release Date:** April 27, 2026

Патч-релиз относительно **v1.1.4**: ноды (регистрация, TLS/URL, enable), панель (форма адреса).

### Исправления и поведение

- **`nodes.enable`:** `SetNodeEnabled` снова пишет **boolean** в PostgreSQL (после миграции **0033** передача `0`/`1` ломала выключение ноды: `encode 0 … for bool`).
- **Регистрация ноды:** запись в БД **только если прошёл health check**; при ошибке проверки запись **удаляется** (нет «висячих» нод после неудачной регистрации).
- **Адрес и TLS (pairing):** для **pairing** URL нормализуется к **`https://`** (воркер с SECRET_KEY слушает TLS+mTLS). Для **legacy**-нод учитывается **`use_tls`**: при `UseTLS == false` используется **plain HTTP** без принудительного апгрейда схемы.
- **`check-connection`:** проба с тем же режимом, что и регистрация (**pairing + TLS**), чтобы результат совпадал с реальным add.
- **Форма нод (UI):** для хоста без `http://`/`https://` по умолчанию **`http`** при выключенном TLS; в мастере добавления по умолчанию **`useTls: false`**; убран неиспользуемый **`legacyAuth`** из сборки адреса.
