## Release v1.2.0

**Release Date:** April 27, 2026

Минорный релиз относительно **v1.1.9**: доработки Hysteria2 и учета трафика, улучшения клиента/нод в панели, live-статусы и расширенная локализация RU.

### Hysteria2 и backend

- **Hysteria2 в панели и подписке:** стабилизация потока после откатов/переапплаев, восстановление корректного per-user учета.
- **Трафик клиентов:** исправлено сопоставление hysteria user stats с email клиента, включая кейсы с несколькими inbound-ами.
- **Xray policy/stats:** выравнивание включения per-user статистики для корректных online/traffic метрик.

### Панель (UI)

- **Clients:** переработана модалка клиента (блок трафика, статус/активность, действия), добавлена относительная активность вида «был N мин назад».
- **Nodes:** добавлен показ online users по нодам и сопутствующие UI-улучшения статистики.
- **Inbounds:** улучшения UX редактирования и русификации action-кнопок/подсказок.

### Локализация

- **RU coverage:** закрыты пропуски переводов в модалках клиентов и inbounds (кнопки действий, подсказки, статусы, подписи блоков).

### Репозиторий

- Удалены временные локальные тестовые артефакты Hysteria2 из `scripts/`, использовавшиеся только для e2e-отладки.

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.2.0`** / **`latest`** и пересоздайте контейнеры панели/ноды.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
