## Release v1.1.1

**Release Date:** April 26, 2026

Патч-релиз относительно **v1.1.0**: исправления совместимости, доработки подписок и UI, уточнения в документации по Docker/Watchtower.

### Исправления и совместимость

- **WireGuard и Xray:** при сборке конфига для ядра адреса интерфейса приводятся к маскам **/32 (IPv4)** и **/128 (IPv6)** — как ожидает Xray 26+. Старые записи в БД (например `10.8.0.1/24`) больше не ломают применение инбаунда.
- **Подписки:** для протокола WireGuard корректно формируется выдача в подписке; для `getLinkWithClient` учтён сценарий **без привязанного ClientEntity** (поиск пира по email в JSON инбаунда).
- **Инбаунды (бэкенд):** для WireGuard настройки **`settings` не пересобираются из клиентов панели** — peers и ключи остаются в серверном JSON инбаунда, без лишнего слияния с моделью клиентов.

### Панель (Next.js)

- Экран **инбаундов** и конструкторы **Xray** (в т.ч. outbounds, simple core, шаблоны): правки UX и согласованности с новыми протоколами/полями.
- Обновления **локализаций** (panel locales + TOML переводы веба).

### Документация

- В **README** (RU/EN/FA) и комментариях **`docker-compose.yml`** добавлено пояснение: при **`build:`** локальное имя образа (например `sharx-code-sharx`) **не является путём в Harbor** — Watchtower/UI-обновление должны использовать **`image:`** из реестра и `docker login`.

### 📦 Upgrade from v1.1.0

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.1.1`** / **`latest`** и пересоздайте контейнеры (или дождитесь Watchtower, если уже настроен Harbor-`image:`).
3. При необходимости проверьте **WireGuard**-инбаунды после рестарта Xray.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
