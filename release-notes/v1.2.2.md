## Release v1.2.2

**Release Date:** April 28, 2026

Патч-релиз относительно **v1.2.1**: настраиваемые интервалы опроса нод (multi-node), адаптивный health при недоступности ноды и правки backend.

### Панель (UI)

- **Настройки (multi-node):** интервал сбора статистики с воркеров (по умолчанию **3** с), интервал проверки **online** (по умолчанию **15** с) и ускоренный интервал при **error / offline / unknown** (по умолчанию **5** с) до восстановления; значения в БД, перезапуск панели не обязателен.

### Multi-node и backend

- **Расписание:** джобы тикают раз в секунду, реальная частота задаётся настройками; health — по ноде (`lastCheck`), stats — один общий цикл на интервал.
- **Overlap:** при незавершённом предыдущем запуске тик пропускается (debug), без накопления параллельных запросов.
- **GET `/api/v1/stats`:** `EOF`, connection reset, broken pipe и схожие ошибки учитываются как retryable в существующей схеме повторов.

### 📦 Upgrade from v1.2.1

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Подтяните образы **`1.2.2`** / **`latest`** и пересоздайте контейнеры панели/ноды.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```

Полный список изменений: `git log v1.2.1..v1.2.2` / `git diff v1.2.1 v1.2.2`.
