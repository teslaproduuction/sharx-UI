## Release v1.3.0

**Release Date:** May 5, 2026

Релиз относительно **v1.2.9**: гибкая **многострочная подписка** (режимы адресов Host относительно нод, порядок строк, публикуемые endpoint’ы), **поля Host для переопределения TLS/транспорта в URI подписки**, уточнённая семантика применения этих оверрайдов **по строкам** (строка CDN/Host и external proxy — да; строки нод из prepend/append — настройки inbound), обновления **backend/API/панели**, модальные **глобальные справки** по разделам Ноды / Подключения / Пользователи / Хосты с полным **i18n**, правки **документации API**.

### New Features

- **Хосты и подписка (multi-node + CDN)**
  - Колонка **`subscription_apply_mode`** у Host: `replace` | `prepend` | `append` — как адрес(а) Host смешиваются с адресами нод в текстовой подписке.
  - На уровне привязки **inbound ↔ node**: порядок в подписке (`sort_order`), переопределение публикуемого адреса/порта, флаг участия в подписке, суффикс remark для строки.
  - На уровне **client ↔ inbound**: `sort_order` для порядка inbounds в выдаче подписки.
  - **Миграция `0037_subscription_link_flex.sql`**.

- **Host: TLS и транспорт в ссылках подписки (`subscription_*`)**
  - Поля: SNI, HTTP Host / authority, path / serviceName, ALPN, fingerprint (`fp`), `allow_insecure` (nullable — наследование из inbound).
  - **Миграция `0038_host_subscription_overrides.sql`**; модель, persistence и API панели для Host.
  - В генераторе подписки (**VMess / VLESS / Trojan / Shadowsocks**, JSON-подписка и связанные пути): оверрайды применяются **только** к строкам, построенным из адреса Host (и к строкам **external proxy** при наличии привязанного Host), а не ко всем строкам инбаунда при режимах prepend/append.

- **Панель: раздел Hosts**
  - Редактирование в **Drawer**, вкладки «Основное» / «Расширенное», выбор inbound’ов, блок **Subscription endpoints (nodes)** с обновлением списка и сохранением привязок, подсказки по полям TLS (`subscriptionTlsSectionHint`).

- **Панель: справка по разделам**
  - Компонент **`SectionHelpModal`**: кнопка «?» в шапке страниц **Ноды**, **Подключения (Inbounds)**, **Пользователи**, **Хосты** → модальное окно с контекстным текстом из i18n.
  - Переводы ключей `pages.help.*`, `pages.{nodes,inbounds,clients,hosts}.helpModal*` во всех поддерживаемых TOML-локалях и перегенерация `panel/public/locales/*.json`.

### Improvements

- **`API.md`**: уточнено описание полей Host `subscription*` — область действия (строки Host / external proxy против строк нод).
- Внутри подсистемы подписки: клонирование maps/query-параметров перед применением оверрайдов Host по строкам, флаг **`ApplyHostSubscriptionOverrides`** у адресов для текстовых ссылок.

### Fixes

- Сохранена согласованность порядка и видимости строк подписки с новыми колонками маппингов (при откате БД без миграций возможны ошибки — см. раздел обновления).

### 📦 Upgrade from v1.2.9

1. Сделайте **бэкап** базы данных и `.env`.
2. Поднимите панель с версией **`1.3.0`** (образы **`1.3.0`** / **`latest`**) — при старте должны примениться миграции **`0037`** и **`0038`**.
3. После старта проверьте:
   - **Хосты** — создание/редактирование, режим адресов подписки, расширенные TLS-поля и список subscription endpoints;
   - **Подписка клиента** — состав строк (replace/prepend/append), что строки нод не перезаписаны TLS с Host там, где ожидаете настройки inbound;
   - **Иконки «?»** на страницах Ноды / Подключения / Пользователи / Хосты.

services:
