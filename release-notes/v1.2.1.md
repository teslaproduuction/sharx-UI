## Release v1.2.1

**Release Date:** April 27, 2026

Патч-релиз относительно **v1.2.0**: скорость клиентов в панели, блокировка сессий по IP, усиление HWID-блокировки и сопутствующие правки Xray/ноды.

### Панель (UI)

- **Clients / Speed:** последняя скорость из коллектора трафика попадает в список клиентов и WebSocket; для Hysteria fallback добавлен расчёт скорости; отображение в **Mbps** вместо «сырых» b/s.
- **Clients / Sessions:** API для добавления IP в пер-клиентный blocklist сессий (`POST /panel/client/sessions/block/:id`).

### Подписка и безопасность

- **Subscription:** отказ выдачи подписки, если IP запроса в blocklist клиента (сессионная блокировка).
- **HWID:** учёт админ-блокировки устройства при регистрации/проверке и в подписке; в ответах API для списка HWID добавлено поле **`blocked`** (удобство UI); лимит устройств не считает записи с `blocked_at`.

### Backend и Xray

- **Трафик / скорость:** in-memory снимок скоростей для панели; merge в `ClientToCardView` и broadcast клиентов; расчёт speed в `applyHysteriaTagDeltas` (Hy2 fallback).
- **Session IP routing:** правила маршрутизации (user + source → outbound `blocked`), hot-apply через RoutingService на локальном core и на воркерах (multi-node), персист в конфиг ноды где применимо.
- **Конфиг API:** автоматическое добавление `RoutingService` в `config.api.services`, если отсутствует (`EnsureAPIServicesRoutingService`).

### База данных

- Миграция **`0034_client_blocked_session_ips`**: таблица `client_blocked_session_ips` (уникальность по паре client_id + ip).

### 📦 Upgrade from v1.2.0

1. Сделайте **бэкап** PostgreSQL и `.env`.
2. Обновите образы на **`1.2.1`** / **`latest`** и перезапустите панель и ноды (нода должна быть той же минорной линии, что ожидает панель для session-ip API).
3. Убедитесь, что миграции применились (таблица `client_blocked_session_ips`).

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```

Полный список изменений: `git log v1.2.0..v1.2.1` / `git diff v1.2.0 v1.2.1`.
