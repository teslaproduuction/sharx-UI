## Release v1.4.1

**Release Date:** May 28, 2026

Релиз **v1.4.1** относительно **v1.4.0**: патч-релиз с критическим исправлением обфускации **salamander** в Hysteria2 — конфигурация формировалась в неправильном JSON-формате и xray не мог прочитать пароль. Дополнительно обновлена документация API.

Миграций БД **нет**.

---

### Bug Fixes

- **Hysteria2 salamander obfs: неверная структура конфига — xray игнорировал пароль и падал с `PSK must be at least 4 bytes`** (`web/service/xray_hysteria_tls.go`, `panel/lib/inboundDefaults.ts`, `panel/lib/xrayMonacoSnippets.ts`)
  - **Причина:** xray-core читает каждый элемент `finalmask.udp[]` через `JSONConfigLoader` со схемой `{"type": "...", "settings": {...}}`. Если ключа `settings` нет, xray подставляет `{}` и пароль оказывается пустой строкой → PSK < 4 байт → краш. Панель при этом писала пароль на верхний уровень объекта (`{"type":"salamander","password":"..."}`) — xray его не видел независимо от длины.
  - **Бэкенд** (`patchHysteriaObfs` в `web/service/xray_hysteria_tls.go`):
    - Чтение пароля изменено с `m["password"]` на `m["settings"]["password"]`, с фолбэком на `m["password"]` для обратной совместимости с конфигурациями, сохранёнными до этого фикса.
    - Запись нормализована до канонической схемы `{"type":"salamander","settings":{"password":"..."}}` — старые инбаунды из БД автоматически исправляются при следующем применении конфига без пересохранения.
  - **Фронтенд** (`buildHysteriaStreamSettingsFromForm` в `panel/lib/inboundDefaults.ts`): writer теперь эмитит `{type:"salamander", settings:{password:...}}`.
  - **Фронтенд** (reader `parseHysteriaStreamSettings`): читает из `settings.password`, с фолбэком на легаси `password` — форма редактирования корректно загружает старые записи.
  - **Сниппет** (`panel/lib/xrayMonacoSnippets.ts`): Monaco-сниппет `hysteria2+salamander` обновлён до правильной схемы.
  - **Следствие**: ошибка воспроизводилась на *любом* пароле — даже 9-символьный `FjgnUs7fa` вызывал краш, так как xray получал `password: ""`.

### Improvements

- **Документация API** (`web/docs/API.md`)
  - Добавлена note в раздел **Inbounds** с пояснением семантики поля `email` в контексте переименования (`email` → `name`) из v1.3.9: `email` в endpoint-ах inbound API — это Xray-ключ (`settings.clients[].email`), а не поле `name` панельного клиента.
  - Обновлены описания параметров `email` в 6 endpoint-ах (`getClientTraffics`, `getClientIps`, `clearClientIps`, `delClient`, `resetClientTraffic`, `updateClientTraffic`) — добавлена расшифровка «Xray client key».
  - Раздел **Clients** (`/panel/client`): `GET /panel/client/list` и `GET /panel/client/get/{id}` задокументированы как возвращающие расширенный `ClientCardView` (поля `activeHwidCount`, `inbounds`, subscription links, `isOnline`).
  - Поле `email` → `name` в примерах ответов и схемах (`/panel/client/list`, `/panel/client/add`, group clients, schema section).
  - Удалено устаревшее поле `limitIp` из примеров и таблиц параметров; добавлены `ipLimitEnabled` / `maxIPs` (из v1.3.9).
  - `lastOnline` задокументирован как миллисекунды (не секунды).
  - `/panel/client/sessions/{id}`: добавлено поле `blockedSessionIps` в пример ответа и таблицу полей; `email` → `name` в response.
  - `/panel/client/add`: параметр `email` → `name`, пометка «immutable after creation».
  - Node API schema: `email` → `name`, удалён `limitIp`, добавлены `ipLimitEnabled` / `maxIPs`.

### Operational Notes

- Обновление затрагивает только **панель** (`sharx`) — нода (`sharxnode`) обновления не требует.
- Существующие инбаунды с salamander **не требуют пересохранения**: бэкенд нормализует конфиг при следующем применении (рестарт панели или ноды). После пересохранения через UI запись в БД также будет в корректном формате.
- Саб-ссылки Hysteria2 с salamander начнут корректно включать `obfs=salamander&obfs-password=...` после применения исправления.

### 📦 Upgrade from v1.4.0

1. Бэкап `.env` (миграции БД не требуются).
2. Обновите образ **`sharx`** до версии **`1.4.1`** / **`latest`**. Образ **`sharxnode`** обновления не требует.
3. Проверьте:
   - Hysteria2-инбаунд с salamander: xray стартует без ошибок `PSK must be at least 4 bytes`.
   - Саб-ссылка клиента содержит `obfs=salamander&obfs-password=<пароль>`.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
