## Release v1.1.0

**Release Date:** April 26, 2026

### ✨ New Features

#### Nodes and pairing
- **Panel-wide node pairing** (`panel_pairing`): one shared `SECRET_KEY` material for all nodes, verification via mTLS and JWT; legacy per-node secrets kept for compatibility.
- **Node enable / disable**: disabled nodes are skipped for health checks, stats collection, and Xray config push.
- **Node geography**: approximate geolocation from egress IP, with geo map UI.
- **Node statistics**: per-node and aggregate views, **client traffic per node** (`client_node_traffics` matrix).
- **Xray process state** on each node: running / stopped / error / unknown (as seen by the panel).
- **Node JWT / pairing helpers** and normalized auth flow toward `pairing` (SECRET_KEY + JWT + mTLS).

#### Subscription and public pages
- **Subscription page builder (SPA)**: stored configs, routing profiles, JSON templates, branding, preview, and **SharX subpage** defaults for a consistent public subscription experience.
- **Provider ID** delivery method already in DB (`subProviderIDMethod`); this release is aligned with the new subscription stack.

#### Xray and protocols
- **Hysteria / Hysteria2** inbounds: protocol index, TLS/QUIC-related handling (`xray_hysteria_tls` and related logic).
- **Xray config UX**: template editor stepper, DNS / outbounds / routing builders, Monaco-based JSON with schemas for safer editing.

#### API and operations
- **API access tokens (JWT)**: use the REST API without session cookies; manage tokens in the panel.
- **Docker / Watchtower**: optional automatic image updates for `sharx` and related services (env-driven).
- **Connection drop (conndrop)**, **geoip** utilities, and **config pull / geo push** on the node side for geography and operability.
- **Tg bot**: large refresh — runtime config, health notifications with hysteresis, proxy URL handling, 2FA/TOTP where applicable.

#### Panel and i18n
- **UI refactor** (Next.js app shell, layout, header/app meta, Telegram nav): cleaner navigation, clients statistics routes, node geography/statistics subpages.
- **New / expanded locales** for the panel and for subscription-related web translations (e.g. ar, es, id, ja, pt, tr, vi, and Toml translation files for multiple languages).

#### Documentation
- **READMEs (RU/EN/FA)**: node workflow via the web UI, update flows, Watchtower where relevant; repository media cleanup (legacy screenshots removed from this tree).

### 📦 Upgrade Instructions

1. **Back up** your PostgreSQL database and panel settings.
2. **Pull** new images and **recreate** containers (or rely on **Watchtower** if you already use it).
3. Update your `docker-compose.yml` to pin or track `1.1.0` / `latest` as you prefer.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```

4. Re-pair or verify **node connectivity** if you changed pairing; check **node enabled** and **geography** after nodes restart.
5. For **API token** or **builder**-driven subscription URLs, re-check external clients and reverse-proxy rules if you customized paths.

For installation from the [SharX](https://github.com/konstpic/SharX) quick-start, see the project README; nodes are managed from the **web UI**.
