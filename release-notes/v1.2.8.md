## Release v1.2.8

**Release Date:** April 30, 2026

Фича-релиз относительно **v1.2.7**: полная переработка страницы подписки клиента — новый UX инструкции по установке с таймлайном шагов, авто-определение платформы по User-Agent, три независимых настройки вида (платформа / приложение / шаги), тогл скрытия приложений, зашифрованные QR-коды per-app, а также переводы страницы на 13 языков.

### New Features

- **Страница подписки: новый flow Инструкции по установке**
  - Убран дублирующий блок «Добавить приложение» — кнопки deep link теперь встроены прямо в шаги инструкции.
  - Вместо карточки с компактным списком шагов — полноценный **вертикальный таймлайн**: каждый шаг оформлен отдельной карточкой с заголовком, описанием и кнопкой действия inline.
  - Каждый шаг несёт своё действие: шаг 1 — Install/Download, шаг 2 — Add subscription + QR, шаг 3 — Connect (без кнопки).

- **Авто-определение платформы по User-Agent**
  - При открытии страницы подписки платформа (iOS, Android, Windows, macOS, Linux, Android TV) автоматически выбирается по `navigator.userAgent`.
  - Если определённая платформа не включена в конфиге — выбирается первая из списка как fallback.

- **Три независимых настройки вида блока Инструкции (конструктор)**
  - **Platform selector** — `Tabs` / `Dropdown` / `Pills` / `Accordion`
  - **App selector** — `Chips` / `List` / `Dropdown`
  - **Steps layout** — `Timeline` / `Numbered` / `Plain`
  - Все три оси комбинируются независимо (4 × 3 × 3 = 36 вариантов). Старое поле `style` сохранено для обратной совместимости.

- **Тогл скрытия приложения без удаления**
  - В редакторе каждого приложения внутри платформы добавлен Switch — можно временно скрыть приложение из публичной страницы, сохранив его настройки в конфиге.
  - Скрытое приложение отображается полупрозрачным с пометкой «hidden».

- **Зашифрованный QR per-app**
  - QR-кнопка перемещена из заголовка приложения прямо на шаг «Add subscription» — inline рядом с кнопкой добавления.
  - QR генерируется с **зашифрованной** ссылкой (`happ://crypt4/…` / `v2raytun://crypt/…`), если для данного приложения включён E2E deep link.
  - При обычной ссылке — QR показывает обычный deep link, fallback — subscription URL.

- **Бейдж E2E в заголовке приложения**
  - Когда для выбранного приложения используется зашифрованный deep link, рядом с названием показывается бейдж `E2E`.

### Improvements

- **Конструктор: блок «Добавить приложение» убран из списка новых блоков**
  - Блок `add-to-app` больше не предлагается при добавлении нового блока (дублировал функциональность Инструкций).
  - Существующие конфиги с `add-to-app` продолжают работать — рендер и редактор сохранены для обратной совместимости.

- **Переводы страницы подписки (13 языков)**
  - Переведены дефолтные шаги инструкции: «Установить приложение», «Добавить подписку», «Подключиться» — с подстановкой имени приложения `{{app}}`.
  - Добавлены переводы всех новых ключей конструктора (`platformView`, `appView`, `stepsView` и их значения).
  - Языки: EN, RU, UK, ZH, TW, FA, AR, TR, ES, PT, ID, VI, JA.

### 📦 Upgrade from v1.2.7

1. Сделайте **бэкап** базы данных и `.env`.
2. Подтяните образы **`1.2.8`** / **`latest`** и пересоздайте контейнеры панели.
3. После старта проверьте:
   - страницу подписки клиента — платформа должна определяться автоматически;
   - конструктор страницы подписки — блок Инструкции по установке теперь имеет три раздельных селектора вида.

> ⚠️ Существующие конфиги страниц подписки мигрируются автоматически: новые поля `platformView`, `appView`, `stepsView` имеют дефолтные значения и не требуют ручного обновления.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
