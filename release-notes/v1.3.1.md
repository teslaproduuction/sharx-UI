### Новое в v1.3.1

#### Пользовательские тексты в подписке (custom remarks / placeholders)

- В конфигурации v2 страницы подписки добавлены **`showCustomRemarks`** (по умолчанию включено; `nil` трактуется как «вкл» для обратной совместимости) и блок **`customRemarks`**: списки строк для разных ситуаций.
- При включённых кастомных заметках текстовая подписка вместо «пустого» или жёстко заблокированного ответа может вернуть **фиктивные строки VLESS** (`vless://` на `0.0.0.0:1` с UUID-заглушкой): клиенты показывают **fragment** как название узла (тот же приём, что и в Remnawave).
- **JSON-подписка:** при срабатывании лимита HWID и настроенных строках — минимальный JSON с **blackhole**-outbounds, теги совпадают с текстами заметок; заголовок трафика сохраняется по клиенту, где применимо.
- **Категории текстов (с дефолтами в коде и в панели):**
  - **ExpiredUsers** — срок подписки истёк;
  - **LimitedUsers** — исчерпан трафик (лимит);
  - **DisabledUsers** — пользователь отключён (`enable = false`), по `sub_id` до основного запроса активного клиента;
  - **EmptyHosts** — нет inbounds/строк подписки после резолва;
  - **HWIDMaxDevicesExceeded** — превышен лимит устройств / админская блокировка HWID (для текстовой подписки — placeholder-строки; для JSON — уведомление как выше);
  - **HWIDNotSupported** — поле есть в конфиге и редакторе (сид для будущего использования); **в текущей генерации текстовой/JSON подписки не используется**.
- Backend: `web/service/sharx_custom_remarks.go` (дефолты, merge, `EffectiveCustomRemarks`), `sub/subscription_placeholder.go` (`subscriptionPlaceholderLines`, `jsonSubscriptionNoticeBody`, `trafficFromClientEntity`, классификация ошибок HWID).
- Панель: компонент **`CustomRemarksEditor`** в **`SubscriptionBuilder`**; схема и дефолты в `panel/lib/sharxSubpageConfig.ts`.

#### HWID: уникальность только по клиенту панели

- Убрана проверка «этот **hwid** уже привязан к **другому** `client_id`». Один и тот же аппаратный идентификатор может быть учтён у **нескольких** клиентов (например, две учётки одному человеку).
- Уникальность по-прежнему обеспечивается парой **`(client_id, hwid)`**; лимит устройств **`MaxHWID`** и админская блокировка строк HWID остаются **в рамках каждого клиента**.
- Удалены `ErrHWIDUsedByAnotherClient` и ветка обработки в `registerHWIDFromRequest` / подписке.

#### Прочее

- Проход **`gofmt`** по затронутым пакетам (выравнивание импортов и пробелов без изменения логики).

### Улучшения и поведение

- Для **отключённого** клиента: при непустом списке `DisabledUsers` возвращаются placeholder-строки и корректный трафик из сущности клиента; иначе — прежняя ошибка подписки.
- Для **истёкшего по времени / по трафику** активного клиента: при `showCustomRemarks` и непустых списках — placeholder вместо полной выдачи узлов (поведение задаётся конфигом конструктора подписки).

### 📦 Upgrade

1. Бэкап БД и `.env`.
2. Образы версии **`1.3.1`** (или `latest` после публикации). **Миграции не требуются** (изменений в `database/migrations/` для этого релиза нет).
3. Проверить:
   - конструктор подписки в панели — блок custom remarks и переключатель показа;
   - сценарии: отключённый клиент, истёкший срок, исчерпанный трафик, нет хостов, лимит HWID (текст и при необходимости JSON);
   - два клиента с одним и тем же HWID (если используете такую схему).

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
