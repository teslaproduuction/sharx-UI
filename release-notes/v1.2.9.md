## Release v1.2.9

**Release Date:** May 5, 2026

Релиз относительно **v1.2.8**: поток логов панели переведён на **SSE** вместо WebSocket (устранение фризов UI), система **контекстных подсказок** с i18n, обновлённая визуализация **статусов нод и Xray** в таблице, правки **PageHeader / статистики / карты гео**, новый **Database Inspector** в настройках, расширенный **i18n-пайплайн** и обновление **bundled Xray до v26.5.3**, доработки **сессий клиента** и покрытия локалей.

### New Features

- **Поток логов: WebSocket → SSE**
  - Отдельный endpoint `GET /api/server/logs/stream` с батчингом (~250 ms), фильтрацией по уровню и источнику на клиента.
  - WebSocket-хаб больше не транслирует логи — только статус, трафик и уведомления.
  - Фронт: хук `useLogStream` (EventSource + экспоненциальный backoff при обрыве).
  - Модал логов: ширина ~95vw, виртуализация через `react-window`, буфер до 2000 строк, кнопки Pause / Resume / Clear / Export, индикатор Live / Paused, цветные бейджи уровней.

- **Контекстные подсказки (Help)**
  - Компонент `HelpTooltip` (иконка справки) рядом с подписями полей; режимы tooltip и модального окна.
  - Тексты только из i18n (`help.*`), без захардкоженных строк.
  - Встроено в разделы: Settings (строки таблиц), Clients (секции формы), Nodes (формы добавления/редактирования), блок информации на публичной странице подписки.

- **Иконки статуса нод и Xray в таблице**
  - Колонки статуса ноды и состояния Xray переведены на **IconTile**-бейджи: Lucide-иконка + цветная подпись (изумрудный / янтарный / розовый / muted по смыслу).
  - Подпись статуса по наведению показывается во всплывающей подсказке; в ячейке остаётся компактная «плитка».

- **Database Inspector (Настройки → БД)**
  - **Backend:** `DbInspectorService` — список таблиц, introspection схемы, постраничные строки с поиском и сортировкой, параметризованные **CREATE / UPDATE / DELETE**; REST под `/panel/db/*`, защита `checkLogin`, регистрация в `XUIController`.
  - **Frontend:** маршрут `/panel/db-inspector`, сервис `DatabaseService`, хуки `useDatabaseTables` / `useTableData`, боковая панель таблиц (desktop + mobile drawer), `EditableCell` (string / number / boolean / date / json), `TableViewer` (сортировка, фильтр, пагинация, inline CRUD, модалка схемы), предупреждение об опасности прямых правок в БД.
  - **i18n:** новые ключи для инспектора и связанных подсказок во всех поддерживаемых локалях.

- **Инструменты локализации и синхронизация TOML**
  - Скрипт `check-locales.mjs` для проверки покрытия ключей.
  - Расширен `gen-locales.mjs` (плейсхолдеры i18next, множественное число).
  - `sync-toml-from-en.mjs` и npm-скрипт `sync-toml` для выравнивания не-EN локалей с английским шаблоном.
  - Расширен `translate.en_US.toml`, перегенерированы JSON локалей, UI переведён на `t()` там, где добавлялись строки.

- **Xray по умолчанию v26.5.3**
  - Файл `data/xray-version.txt` с версией bundled Xray.
  - `DockerInit.sh`, образ ноды и комментарии в Docker-артефактах приведены в соответствие; колонка `nodes.xray_version` (кэш версии воркера) задокументирована в миграции комментарием с примером `26.5.3`.

### Improvements

- **PageHeader:** иконка в одной строке с заголовком (flex row), в том числе на мобильных — без «стэка» иконки над текстом.
- **NodesStatisticsPage:** пояснительный текст перенесён в `description` у `PageHeader`.
- **NodesGeographyPage:** тулбар легенды адаптирован под узкие экраны (кнопка на всю ширину на mobile).
- **Клиентские сессии:** при ответе ноды «online, записей нет» показывается пустое состояние; развёрнутые сообщения об ошибках для прочих сбоев ноды.
- **Локали RU/EN и др.:** ключи для настроек, DB Inspector, модалки сессий, сохранения inbounds (back/next) и связанных экранов.

### Fixes

- **ESLint:** удалён неиспользуемый хелпер в `TableViewer`, убран устаревший disable `exhaustive-deps`, поправлены зависимости `useCallback` на `GroupsPage`.
- **HelpTooltipTrigger:** убран неиспользуемый проп `helpKey` (предупреждение линтера).

### 📦 Upgrade from v1.2.8

1. Сделайте **бэкап** базы данных и `.env`.
2. Подтяните образы **`1.2.9`** / **`latest`** и пересоздайте контейнеры панели и нод (при необходимости — отдельно обновите `sharxnode` для Xray **26.5.3**).
3. После старта проверьте:
   - **Логи** (модал в панели) — поток должен идти без подвисаний интерфейса;
   - **Настройки → Database Inspector** — доступ только под учётной записью панели; убедитесь, что политика доступа к этому разделу вам подходит;
   - **Ноды** — бейджи статусов и отображение версии Xray при необходимости;
   - **Страницы статистики / географии** — вёрстка на мобильном.

> ⚠️ **Database Inspector** позволяет выполнять произвольные изменения в таблицах — используйте только осознанно на копиях или в обслуживаемых окружениях.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
