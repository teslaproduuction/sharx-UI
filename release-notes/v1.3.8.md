## Release v1.3.8

**Release Date:** May 15, 2026

Релиз относительно **v1.3.7**: два исправления и одно улучшение. **Hysteria / Hysteria2** в мульти-нодовых конфигурациях теперь генерирует корректные ссылки для каждой ноды (раньше всегда подставлялся адрес панели). Устранено **двойное появление строк Xray** в UI-логах. Добавлен управляемый **Emit-гейт** уровня логирования — debug-шум из Xray и нод больше не заливает дашборд, если оператор выбрал `info` / `warning` / `error`.

Миграций БД **нет**.

### Bug Fixes

- **Hysteria2 / Hysteria — неверные адреса в мульти-нодовых подписках** (`sub/subService.go`, `sub/subJsonService.go`)
  - **Было:** `hysteriaLinkForAuth` всегда использовал `s.address` (адрес панели) вместо адресов нод, из-за чего при наличии нескольких нод все hysteria/hysteria2-ссылки указывали на панель.
  - **Стало:** функция итерируется по результату `getAddressesForInbound` — так же, как уже делали `genVlessLink` / `genTrojanLink`. Для каждой ноды генерируется отдельная ссылка с правильным адресом, портом и суффиксом ремарки.
  - **JSON-подписка** (`subJsonService.go`): `getConfig` аналогично заменяет захардкоженный `externalProxy` (всегда указывал на хост панели) на список адресов из `getAddressesForInbound`. Hysteria2 в JSON-формате теперь тоже получает корректные хосты/порты для каждой ноды.

- **Дублирование строк Xray в UI-логах** (`web/web.go`)
  - **Было:** `XrayLogTailJob` тейлил файлы `xray-access.log` / `xray-error.log` каждую секунду и отправлял строки в буфер. Одновременно `xray.LogWriter.Write → logger.Emit` уже перехватывал stderr Xray и делал то же самое — каждая строка появлялась в UI дважды.
  - **Стало:** `XrayLogTailJob` отключён (закомментирован). Единственный источник Xray-логов для in-memory буфера, Loki и SSE-стрима — `LogWriter.Write → logger.Emit`.

### Improvements

- **Управляемый Emit-гейт уровня логирования** (`logger/logger.go`, `web/service/setting.go`, `node/xray/manager.go`)
  - Добавлена функция `SetMinEmitLevel(level string)` с RW-мьютексом. Устанавливает минимальный уровень, ниже которого записи **не попадают** в in-memory буфер UI, Loki и node→panel push-пайплайн. Консольный и файловый бэкенды по-прежнему управляются своим уровнем из `InitLogger`.
  - `UpdateAllSetting` вызывает `SetMinEmitLevel` при каждом сохранении настроек, синхронизируя гейт с выбором оператора (`panelLogLevel` / env var). При включённом Grafana/Loki уровень принудительно выставляется в `debug` для полной передачи логов в Loki.
  - `ensureXrayLoggingDefaults` (панель) и `ensureNodeXrayLoggingDefaults` (нода) больше **не перезаписывают** `loglevel` в конфиге Xray значением `"debug"`. Значение сохраняется, если оно валидно (`debug` / `info` / `warning` / `error` / `none`); при невалидном или отсутствующем — используется `resolveXrayLogLevelFromPanel()`, которая маппит `panelLogLevel` → Xray loglevel (в т.ч. переименовывает `warn` → `warning`, которое Xray не принимает). Безопасный дефолт — `"warning"`.

### Operational Notes

- Обновление только **панели** до **1.3.8** достаточно для исправления дублирования логов и Emit-гейта.
- Для корректных hysteria/hysteria2-ссылок в мульти-нодовых подписках обновите **и панель, и worker-образы** до **1.3.8** (нода передаёт адреса через `getAddressesForInbound`).
- После обновления проверьте настройки логирования в панели: если ранее был выбран уровень `info` или выше, debug-строки Xray перестанут появляться в UI-логах.

   - Клиент с Hysteria2 на нескольких нодах — в подписке должны быть отдельные ссылки с адресами каждой ноды.
